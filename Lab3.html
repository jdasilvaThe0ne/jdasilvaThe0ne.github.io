URDF Overview

The Unified Robot Description Format (URDF) is an XML-based specification used in ROS 2 to describe the physical structure of a robot.

URDF defines:

  • Links (rigid bodies)
  • Joints (connections between links)
  • Visual, collision, and inertial properties

URDF files are typically stored in the urdf/ directory of a ROS 2 package.

Robot Structure

The robot described in this lab is a simple articulated system composed of a base and one or more movable links.

Example URDF snippet: 

<robot name="simple_bot">
  <link name="base_link">
    <visual>
      <geometry>
        <box size="0.4 0.4 0.2"/>
      </geometry>
    </visual>
  </link>
</robot>

Links and Joints

Links represent rigid components, while joints define how those components move relative to one another.

Common joint types include:

  • fixed
  • revolute
  • continuous
  • prismatic

Example revolute joint: 

<joint name="joint1" type="revolute">
  <parent link="base_link"/>
  <child link="link1"/>
  <origin xyz="0 0 0.2" rpy="0 0 0"/>
  <axis xyz="0 0 1"/>
</joint>

TF and TF2

TF is the ROS system responsible for keeping track of multiple coordinate frames over time.

TF2 is the updated and improved implementation used in ROS 2.

Key benefits of TF2:

  • Time-aware transformations
  • Better threading and buffering
  • Improved API consistency
ros2 run tf2_tools view_frames

Visualizing the Robot in RViz2

RViz2 is a 3D visualization tool used to inspect robot models, sensor data, and coordinate frames.

To display a URDF model: 

ros2 run joint_state_publisher_gui joint_state_publisher_gui
ros2 run robot_state_publisher robot_state_publisher simple_bot.urdf

The RobotModel display type must be added in RViz2, with the correct robot_description parameter selected.

Launch Files

Launch files allow multiple ROS 2 nodes to be started together with a single command.

A Python launch file may start:

  • robot_state_publisher
  • joint_state_publisher
  • RViz2

This ensures consistent startup and reduces configuration errors.

RRBot Comparison

The RRBot example is a reference robot used in ROS documentation to demonstrate a two-link revolute manipulator.

Comparison with this lab’s robot:

  • Both use revolute joints
  • Both rely on TF for frame transforms
  • RRBot includes more detailed inertial modeling

Studying RRBot helps reinforce best practices in URDF design.
